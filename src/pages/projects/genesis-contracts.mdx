## Overview

Genesis Contracts is the foundational smart contract layer for the Genesis protocol. It covers token issuance, staking mechanics, and on-chain governance — the pieces that everything else in the ecosystem builds on.

The codebase is written in Solidity and targets EVM-compatible chains. Contracts are organized by domain so each module can be audited, upgraded, or replaced independently.

## Tech Stack

| Layer | Technology |

## Architecture

```text
genesis-contracts/
├── core/           # Token & registry contracts
├── staking/        # Stake, unstake, reward distribution
├── governance/     # Proposals, voting, timelock
└── interfaces/     # Shared ABIs for integrators
```

### Core Module

The core module defines the native token and a central registry that other contracts reference. Integrators read addresses from the registry instead of hardcoding deployments per chain.

### Staking Module

Users lock tokens to earn rewards over epochs. Rewards are calculated per-share to keep gas costs predictable as the staker set grows.

### Governance Module

Proposal creation, voting, and execution run through a timelock. This gives token holders time to react before privileged actions execute on-chain.

## Getting Started

```bash
git clone https://github.com/Metatime-Technology-Inc/genesis-contracts
cd genesis-contracts
forge install
forge test
```

### Local Development

1. Copy `.env.example` to `.env` and fill in RPC URLs.
2. Run `forge test -vv` for the full test suite.
3. Use `forge script script/Deploy.s.sol` for dry-run deployments.

## Key Features

- **Modular layout** — swap or upgrade one module without touching the rest.
- **Registry pattern** — single source of truth for contract addresses across chains.
- **Epoch-based rewards** — predictable staking cycles with on-chain accounting.
- **Timelocked governance** — proposals cannot execute instantly after passing.
