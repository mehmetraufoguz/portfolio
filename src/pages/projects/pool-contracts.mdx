## Overview

Pool Contracts implements the liquidity layer for the Genesis ecosystem. It provides AMM-style pools, LP token accounting, and reward hooks that integrate with the Genesis staking system.

## Tech Stack

| Layer | Technology |
| --- | --- |
| Language | Solidity ^0.8.x |
| AMM Model | Constant-product (x · y = k) |
| LP Tokens | ERC-20 receipt tokens |
| Testing | Foundry |

## Architecture

```text
pool-contracts/
├── pools/          # Pair creation & swap logic
├── rewards/        # Incentive distribution
├── oracles/        # Price feed adapters
└── periphery/      # Router & helper contracts
```

### Pools

Each pool holds two assets and mints LP tokens proportional to liquidity added. Swaps charge a configurable fee that flows to LPs and the protocol treasury.

### Rewards

External reward tokens can be streamed to LP stakers over time. The reward module reads LP balances at epoch boundaries to avoid manipulation within a block.

## Getting Started

```bash
git clone https://github.com/Metatime-Technology-Inc/pool-contracts
cd pool-contracts
forge install
forge test
```

## Key Features

- **Composable pools** — add new pairs without redeploying the factory.
- **Fee tiers** — support multiple fee levels per asset pair.
- **Reward streaming** — plug in incentive programs per pool.
- **Router helpers** — multi-hop swaps and zap-in/zap-out utilities.
