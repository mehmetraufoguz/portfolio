## Why I looked for this

I wanted a contribution graph on the homepage without pulling in a heavy charting library or building SVG math from scratch. [react-github-calendar](https://github.com/grubersjoe/react-github-calendar) wraps the same visual language GitHub uses and ships a small React API.

## Installation

```bash
npm install react-github-calendar
```

## Basic usage

The component only needs a GitHub username. It fetches contribution data client-side and renders an interactive heatmap:

```tsx
import GitHubCalendar from 'react-github-calendar'

export function GitHubContributions() {
  return <GitHubCalendar username="mehmetraufoguz" />
}
```

Because it relies on browser APIs for fetching and interaction, the wrapper component is marked `'use client'` in this site.

## What I liked

- **Familiar UX** — visitors instantly recognize the GitHub-style grid.
- **Small surface area** — one prop for the common case, optional theming when you need it.
- **Dark mode** — the library respects color scheme, which matters on a site with `light dark` toggling.

## Trade-offs

- **Client-only** — not ideal for SSR-first pages without a client boundary; fine for a portfolio widget.
- **External data** — depends on GitHub's contribution API; private or sparse profiles show less activity.
- **Bundle size** — heavier than a static image, but much lighter than a general chart library.

## Verdict

For a personal site where the contribution graph is decorative but meaningful, this package hits a sweet spot. I would reach for something custom only if I needed offline data or a non-GitHub source.
